소켓은 소프트웨어로 작성된 추상적인 개념의 통신 접속점이다.
네트워크 애플리케이션은 소켓을 통하여 통신망의 데이터를 송수신한다.

윈도우 소켓은 소켓을 윈도우 환경에서 사용할 수 있게 만든것
Windows Socket을 줄여 윈속(winsock)이라 부른다

윈도우에서 socket을 사용하려면 윈속을 먼저 사용한후에 사용이 가능하다.

서버와 클라이언트로 분류 가능
서버가 먼저 실행된 다음 클라이언트의 접속을 기다려야한다.
클라이언트에서 보내는 패킷에 정보가 있어 IP주소와 포트번호를 알 필요가 없다.

클라이언트( 서버 접속자 및 이용자 )
반드시 서버가 실행된 후 실행되어야한다.
서버에 접속을 시도해야 하고
서버에 접속하기위해서 IP주소와 포트 번호를 알고 있어야한다.

뮤텍스(MUTEX)
공유 리소스에 대해 오직 하나의 스레드만 접근 허용
서로 다른 프로세스에 속한 스레드에도 사용 가능
하나의 프로세스에서 여러 스레드를 사용할 때 사용한다.
어떤 데이터를 사용하고 있는 동안 다른 스레드는 이 데이터를 건드리지 못하게 한다.
데이터를 먼저 사용하고 있는 다른 스레드에서 사용이 끝날 때 까지 대기

CreateMutex()함수를 통해 얻지 않은 Mutex는 서로 다른 프로세스간의 
동기화에 사용 할 수 없기 때문에 크리티컬 섹션과 같다고 할 수 있다.
